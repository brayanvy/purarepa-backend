Autenticación JWT

La API usa JSON Web Tokens (JWT) mediante djangorestframework-simplejwt. La mayoría de endpoints requieren un token de acceso en el header Authorization.

Flujo de autenticación

  1. Obtener token — Enviar credenciales a POST /api/auth/token/. Recibirás access y refresh.
  2. Usar token — Incluir en cada request protegido: Authorization: Bearer <access_token>
  3. Renovar token — Cuando el access expire (24h), usar POST /api/auth/token/refresh/ con el refresh token (válido 7 días).

Ejemplo de uso

bash 
# 1. Obtener token
curl -X POST https://purarepa-api.onrender.com/api/auth/token/ \
  -H "Content-Type: application/json" \
  -d '{"username": "tu_usuario", "password": "tu_password"}'

# Respuesta:
{
  "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

# 2. Usar el token en requests protegidos
curl https://purarepa-api.onrender.com/api/orders/ \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."

Niveles de acceso

Nivel Descripción Endpoints
Público Sin token requerido GET /api/products/, POST /api/auth/token/, POST /api/auth/register/
Autenticado Requiere JWT válido /api/orders/, /api/auth/profile/
ADMIN Requiere role=ADMIN o is_staff=True POST/PUT/DELETE /api/products/, /api/reports/, /api/orders/admin/

Códigos de error

CódigoSignificadoCausa común
200 OK Request exitoso
201 Created Recurso creado exitosamente
400 Bad Request Datos inválidos, stock insuficiente, dirección vacía
401 Unauthorized Token ausente, expirado o inválido
403 Forbidden Token válido pero sin permisos suficientes (requiere ADMIN)
404 Not Found Recurso no existe o no pertenece al usuario
500 Server Error Error interno del servidor

Formato de error estándar

json 
// Error de validación (400)
{
  "delivery_address": ["La dirección de entrega es obligatoria."],
  "items": ["Este campo es requerido."]
}

// Error de autenticación (401)
{
  "detail": "No se han proporcionado credenciales de autenticación."
}

// Error de permisos (403)
{
  "detail": "No tienes permiso para realizar esta acción."
}
