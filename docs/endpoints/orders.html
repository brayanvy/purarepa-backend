📦 Órdenes

Creación y gestión de pedidos con pago contra entrega. Decremento atómico de stock garantizado.

🔒 Todos los endpoints de órdenes requieren JWT. Los clientes solo ven sus propias órdenes; los administradores ven todas.
MétodoEndpointDescripciónAuth
GET /api/orders/ Listar órdenes del usuario (o todas si ADMIN) JWT
POST /api/orders/ Crear nueva orden JWT
GET /api/orders/{id}/ Detalle de una orden JWT
PATCH /api/orders/{id}/ Actualizar estado de la orden ADMIN
POST /api/orders/{id}/upload_proof/ Subir comprobante de pago JWT
GET /api/orders/admin/proofs/ Listar comprobantes pendientes ADMIN
PATCH /api/orders/admin/proofs/{id}/verify/ Aprobar o rechazar comprobante ADMIN
POST /api/orders/ Crear orden JWT

Crea una nueva orden con los ítems del carrito. El proceso es atómico: si algún producto no tiene stock suficiente, toda la operación se revierte y no se modifica ningún stock.

💡 El campo total es calculado automáticamente por el servidor usando el precio vigente de cada producto al momento de la compra. No se puede enviar desde el cliente.

Request body

json 
{
  "payment_method":   "MANUAL",
  "delivery_address": "Calle 45 # 12-30, Barrio Chapinero, Bogotá",
  "items": [
    { "product_id": 1, "quantity": 2 },
    { "product_id": 3, "quantity": 1 }
  ]
}
CampoTipoRequeridoDescripción
payment_methodenum✅MANUAL (contra entrega) | GATEWAY (MercadoPago)
delivery_addressstring✅Dirección completa de entrega. No puede estar vacía.
itemsarray✅Lista de ítems. Mínimo 1 ítem.
items[].product_idinteger✅ID del producto
items[].quantityinteger✅Cantidad solicitada (≥ 1)

Errores posibles

json — 400 dirección vacía 
{
  "delivery_address": ["La dirección de entrega es obligatoria."]
}
json — 400 stock insuficiente 
["Stock insuficiente: Arepa de Choclo"]

Ejemplo cURL

bash 
curl -X POST https://purarepa-api.onrender.com/api/orders/ \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "payment_method": "MANUAL",
    "delivery_address": "Calle 45 # 12-30, Chapinero, Bogotá",
    "items": [
      {"product_id": 1, "quantity": 2},
      {"product_id": 3, "quantity": 1}
    ]
  }'
PATCH /api/orders/{id}/ Actualizar estado de la orden ADMIN

Permite al administrador cambiar el estado de una orden.

Estados válidos

EstadoDescripción
PENDINGPedido recibido, pendiente de confirmación
PAIDPago confirmado
SHIPPEDPedido en camino
CANCELLEDPedido cancelado

Ejemplo

bash 
curl -X PATCH https://purarepa-api.onrender.com/api/orders/42/ \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{"status": "SHIPPED"}'
POST /api/orders/{id}/upload_proof/ Subir comprobante de pago JWT

Permite al cliente subir un comprobante de pago para órdenes con payment_method=MANUAL. Solo se puede subir un comprobante por orden.

bash 
curl -X POST https://purarepa-api.onrender.com/api/orders/42/upload_proof/ \
  -H "Authorization: Bearer <access_token>" \
  -F "file=@/ruta/comprobante.jpg"
