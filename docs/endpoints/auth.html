🔐 Autenticación

Registro de usuarios, obtención y renovación de tokens JWT, y consulta de perfil.

MétodoEndpointDescripciónAuth
POST /api/auth/register/ Registrar nuevo usuario Público
POST /api/auth/token/ Obtener access + refresh token Público
POST /api/auth/token/refresh/ Renovar access token Público
GET /api/auth/profile/ Obtener perfil del usuario autenticado JWT
POST /api/auth/register/ Registrar nuevo usuario Público

Crea una nueva cuenta de usuario con rol CLIENT.

Request body

json 
{
  "username": "maria_garcia",
  "email":    "maria@ejemplo.com",
  "password": "MiPassword123!",
  "phone":    "3101234567"
}
CampoTipoRequeridoDescripción
usernamestring✅Único, sin espacios
emailstring✅Formato válido de email
passwordstring✅Mínimo 8 caracteres, validado por Django
phonestring❌Teléfono de contacto

Respuestas

json — 201 Created 
{
  "message": "Usuario creado correctamente."
}
json — 400 Bad Request 
{
  "username": ["Ya existe un usuario con ese nombre."],
  "password": ["Esta contraseña es demasiado común."]
}

Ejemplo cURL

bash 
curl -X POST https://purarepa-api.onrender.com/api/auth/register/ \
  -H "Content-Type: application/json" \
  -d '{
    "username": "maria_garcia",
    "email": "maria@ejemplo.com",
    "password": "MiPassword123!",
    "phone": "3101234567"
  }'
POST /api/auth/token/ Obtener tokens JWT Público

Autentica al usuario y devuelve un par de tokens JWT. El access token expira en 24 horas; el refresh en 7 días.

Request body

json 
{
  "username": "maria_garcia",
  "password": "MiPassword123!"
}

Respuestas

json — 200 OK 
{
  "access":  "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyX2lkIjoxLCJleHAiOjE3...",
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyX2lkIjoxLCJleHAiOjE3..."
}
json — 401 Unauthorized 
{
  "detail": "No active account found with the given credentials"
}
💡 Guarda el access token en memoria segura (no en localStorage). En Flutter usa flutter_secure_storage.
POST /api/auth/token/refresh/ Renovar access token Público

Genera un nuevo access token usando el refresh token. Úsalo cuando recibas un 401 en cualquier endpoint protegido.

Request body

json 
{
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

Respuesta — 200 OK

json 
{
  "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyX2lkIjoxLCJleHAiOjE3..."
}
GET /api/auth/profile/ Perfil del usuario autenticado JWT

Devuelve los datos del usuario asociado al token JWT enviado en el header.

Respuesta — 200 OK

json 
{
  "id":       1,
  "username": "maria_garcia",
  "email":    "maria@ejemplo.com",
  "phone":    "3101234567",
  "role":     "CLIENT"
}

Ejemplo cURL

bash 
curl https://purarepa-api.onrender.com/api/auth/profile/ \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
